Design Notes

Placement & Rationale

Option A (this mockup)
Standalone section between hero and "How it works". Jurors who land on the homepage can immediately see the entry point. Keeps the hero clean; access code section gets dedicated breathing room.
Option B (alternative)
Inline in the hero itself — a compact input row below the CTA buttons. Saves scroll, but competes with "Experience Demo" and "Explore Admin Panel". Best if jurors are the primary audience.
Implementation cost
The logic is identical to JuryGatePage.handleVerify. Call verifyEntryToken(t), then setJuryAccess(res.period_id), then navigate. A ~40-line component.
